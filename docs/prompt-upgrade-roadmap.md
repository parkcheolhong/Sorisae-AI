### 2️⃣ 인터넷 검색 기반 답변 (새로운 기능)

**현재 수준**: ❌ 미지원

**추가 가능**: ✅ YES (완전히 구현 가능)

#### 아키텍처 옵션

**옵션 A: 경량 웹 검색 통합** (권장 ⭐)

```
사용자 입력
  ↓
Orchestrator: "검색 필요한가?" 판단
  ↓
YES: SerpAPI / Bing Search API 호출
  ↓
검색 결과 → 프롬프트 컨텍스트 추가
  ↓
LLM: 검색 결과 + 학습 데이터 결합
  ↓
최종 답변 + 출처 기록
```

**구현 범위**:

- SerpAPI 또는 Bing Search API 연동
- 질문 유형 분류 (검색 필요 vs 불필요)
- 검색 결과 정제 및 프롬프트 통합
- 신뢰도 표시 (인터넷 기반 vs 추론 기반)

**난이도**: ⭐⭐⭐☆☆ (중간)
**비용**: ~$100-300/월 (검색 API)
**효과**: +40% 현재 정보 답변 정확도
**소요 시간**: 1-2주

---

def _estimate_reply_target_length(message: str) -> str:
    """사용자 입력 길이 기반 목표 응답 길이 추정"""
    normalized = _normalize_chat_message(message).strip()
    char_count = len(normalized)

    elif char_count < 50:
        return "답변 2-4문장 중심"
    elif char_count < 100:
        return "상세 설명 3-5개 단락"
