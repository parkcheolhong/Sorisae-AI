# 본인확인 공급사 사업자 유형 가이드

## 목적

- `개인사업자`와 `법인사업자`를 구분해 공급사 계약, 정산, 운영 전환 조건을 정리합니다.
- 기술 연동과 별개로 실제 상용 운영 가능성을 판단하는 기준 문서입니다.

---

## 1. 개인사업자

### 가능성
- 상용 계약 가능성이 있음
- 공급사별 심사 조건 충족 필요

### 보통 요구 서류
- 사업자등록증
- 대표자 정보
- 서비스 URL / 도메인
- 개인정보처리방침
- 이용약관
- 정산 계좌 정보
- 담당자 연락처

### 과금/정산 확인 항목
- 성공 건 과금 여부
- 실패 건 과금 여부
- 재시도 과금 여부
- 월 기본료
- 테스트 계정 무료 여부
- 세금계산서 발행 가능 여부

### 리스크
- 트래픽이 크면 추가 심사 가능
- 업종/서비스 목적에 따라 제한 가능

---

## 2. 법인사업자

### 가능성
- 가장 일반적인 상용 계약 주체
- 운영 전환 가능성이 가장 높음

### 보통 요구 서류
- 법인 사업자등록증
- 법인명/담당자 정보
- 서비스 설명서
- 개인정보처리방침
- 이용약관
- 정산/세금계산서 정보
- 보안 담당자 또는 운영 담당 정보

### 과금/정산 확인 항목
- 성공/시도/실패 과금 기준
- 월 기본료 및 최소 약정
- 초과 사용 단가
- 익월 청구/선불/후불 구조
- 세금계산서 발행 주기
- 운영/테스트 계정 분리 비용

### 장점
- 운영 심사 통과 가능성이 높음
- 대량 트래픽 단가 협상 가능성 있음

---

## 3. 자연인 개인

### 현실적인 상태
- 개발과 테스트 구조 설계는 가능
- 상용 계약은 어려운 경우가 많음

### 제약 이유
- 개인정보 처리 책임 주체 요건
- 세금계산서/정산 구조 부재
- 운영 서비스 심사 요건 미충족 가능성

### 권장안
- 개인사업자 등록 후 진행
- 또는 법인 명의로 계약 진행

---

## 4. 현재 프로젝트 권장안

- 기술 개발은 계속 진행
- 상용 전환은 `개인사업자` 또는 `법인사업자` 기준으로 준비
- 상품 원가 계산에는 반드시 아래 포함
  - 본인확인 건당 비용
  - 월 기본료
  - 실패/재시도 과금 여부
