## 2. 알람 조건 정의

### 2.1 치명 오류 임계치 (CRITICAL)

| 조건 | 임계치 | 집계 윈도우 | 액션 |
|------|--------|-------------|------|
| HTTP 5xx 비율 | ≥ 5% | 5분 | 즉시 온콜 호출 |
| 결제 실패(`checkout session` 오류) | ≥ 3건 | 5분 | 즉시 온콜 호출 |
| 인증 실패(`401/403`) 급증 | ≥ 50건 | 5분 | 즉시 온콜 호출 |
| DB 연결 불가(`database unavailable`) | 1회 이상 | 1분 | 즉시 온콜 호출 |
| AI 엔진 전체 응답 없음 | 3회 연속 | 3분 | 즉시 온콜 호출 |

- [ ] **2.1** 치명 오류 알람 임계치 설정 완료

### 2.2 경고 수준 임계치 (WARNING)

| 조건 | 임계치 | 집계 윈도우 | 액션 |
|------|--------|-------------|------|
| API 응답 지연 P95 | ≥ 2,000ms | 5분 | Slack #alerts 알림 |
| AI 엔진 응답 지연 P95 | ≥ 10,000ms | 5분 | Slack #alerts 알림 |
| 캐시 웜업 실패(`[WARN] admin capability cache warmup failed`) | ≥ 2회 | 15분 | Slack 알림 |
| 재시도 급증 (워커 재시작) | ≥ 5회 | 15분 | Slack 알림 |
| 4xx 비율 증가 | ≥ 15% | 10분 | Slack 알림 |

- [ ] **2.2** 경고 수준 알람 임계치 설정 완료

### 2.3 알람 집계 윈도우 및 연속 발생 조건

- [ ] **2.3.1** 단발성 오류 노이즈 제거: 동일 알람 조건이 **2회 이상 연속** 충족 시에만 발화
- [ ] **2.3.2** 롤링 윈도우 기준: 5분 / 15분 두 단계 운용
- [ ] **2.3.3** 알람 억제(suppression): 동일 알람 재발화 쿨다운 **10분**
- [ ] **2.3.4** 알람 그루핑: 동일 서비스 내 복수 조건 동시 발화 시 단일 인시던트로 묶음
