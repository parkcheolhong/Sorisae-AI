# 전화 통역 T0 — 타당성·계약 조사 리포트

> **상태:** 🟡 T0 산출물(타당성 리포트). [`TELEPHONY_BRIDGE_DESIGN.md`](TELEPHONY_BRIDGE_DESIGN.md) §4 로드맵의 첫 단계.
> **결론 요약:** 정식 서비스는 **경로 B(서버측 SIP/통신사 브리지)** 로 수렴하되, **번호·트렁크는 직접 등록(기간통신사업)이 아니라 CPaaS/통신사 트렁크 임대**로 시작하는 것을 권장. PoC/데모는 경로 A(인앱 음향 브리지) 병행.
> **이 문서는 의사결정 입력이다 — 계약·법무 확정은 사업 결정 + 전문가(통신 규제/개인정보) 검토 후 진행.** 규제 수치는 공개 법령 기준이며 **계약 전 재확인 필요**로 표기.

---

## 0. T0 목표 (게이트 정의)

T1(미디어 서버 PoC) 착수 전에 답해야 할 4문항:

1. **번호 모델** — 어떤 번호로 착신을 받는가? (070 인터넷전화 / 050 안심번호 / 일반 시내·이동 / 해외 DID)
2. **트렁크 공급** — 직접 등록 vs CPaaS/통신사 임대, 누구와?
3. **비용 구조** — 분당 종단·발신, 번호 월정액, 동시 채널, 미디어 서버 컴퓨트.
4. **규제·동의** — 통역/녹음 고지·동의, 사업자 등록 의무 여부, 개인정보 보존·삭제.

각 문항에 **Go 기준**을 정의하고(§6), 충족 시 T1로 진행한다.

## 1. 번호 모델 옵션

| 모델 | 설명 | 장점 | 제약 | 적합도 |
|------|------|------|------|--------|
| **070 인터넷전화** | VoIP 전용 번호 | 저비용, SIP 친화 | "인터넷전화" 인식·일부 수신거부, 등록 역무 | PoC·국내 베타 ◎ |
| **050 평생/안심번호** | 가상번호(중개) | 개인정보 보호, 발신자 비노출 | 중개사 의존, 기능 제한 | 프라이버시 우선 ○ |
| **일반 시내(02 등)·이동(010)** | 표준 번호 | 신뢰도·수신성 최고 | 발급·이동번호 규제 까다로움 | 정식 서비스(후기) △ |
| **해외 DID(CPaaS)** | 국가별 로컬 번호 | 글로벌 착신, API 즉시 발급 | 한국 번호는 별도 규제, 국제 종단 요금 | 해외 통역 수요 ○ |

> **권장 시작:** 070(국내) + CPaaS 해외 DID(글로벌). 정식 확대 시 통신사 제휴로 일반 번호 검토.

## 2. 트렁크 공급 — 직접 등록 vs 임대

### 2.1 직접 등록 (기간통신사업 — 구 별정통신)
한국에서 인터넷전화 번호를 **직접 발급·운영**하려면 전기통신사업법상 **기간통신사업 등록**(과기정통부)이 필요. *(공개 법령 기준, 계약 전 재확인 필요)*

- **회선설비 미보유 재판매사업**: 납입자본금 **3억원 이상**.
- **회선설비/교환설비 보유사업**: 납입자본금 **30억원 이상**.
- **기술인력**(정보통신기술사·기사 등) + **상설 이용자 보호기구**(전담직원 1명+) 요건.
- 등록증 발급 처리기간 **30일 이내**(적합성평가·주파수 기간 제외).
- 근거: 전기통신사업법 제6·21·22조, 시행령 제8조·[별표1·2].

→ **초기 단계엔 과도.** 자본금·인력·보호기구 부담이 PoC 규모에 부적합.

### 2.2 임대 (권장)
- **국내 통신사 인터넷전화 SIP 트렁크 제휴**(KT/LG U+/SK브로드밴드 등) — 번호·트렁크를 임대받아 우리 미디어 서버에 연결. 직접 등록 의무 회피, 통신사가 망·번호 책임.
- **CPaaS(Twilio/Telnyx 등) SIP Trunk + Programmable Voice** — API로 즉시 번호 발급·종단, 글로벌 확장 용이. 한국 로컬 번호는 공급사별 규제·KYC 상이(확인 필요).

> **결정 권장:** **임대로 시작**(국내=통신사/재판매 제휴, 해외=CPaaS). 직접 등록은 트래픽·매출이 등록비용을 정당화할 때 재검토.

## 3. 비용 모델 (템플릿 + 워크드 예시)

### 3.1 비용 항목
```
월비용 ≈ (번호 월정액 × 번호수)
       + (분당 종단요금 × 통화분)            # 착신 leg
       + (분당 발신요금 × 통화분)            # 통역 합성 주입/콜백 leg (브리지 구조에 따라)
       + (동시 채널/CPS 비용)
       + (미디어 서버 컴퓨트: SFU/B2BUA + STT/MT/TTS GPU)
       + (CPaaS 부가: 녹음·전사 사용 시)
```

### 3.2 CPaaS 종단 단가 (Twilio Elastic SIP Trunking, 공개가 기준)
- **Origination(착신) 시작가**: US 로컬 **$0.0034/min**, US 토프리 **$0.0130/min**. *(지역·번호유형별 상이, 계정별 실단가는 Pricing API로 조회)*
- 한국 종단/번호 단가는 별도 — **공급사 견적 필요**(확인 필요).
- 출처: Twilio SIP Trunking Pricing(US). 단가는 변동·계정 할인 영향.

### 3.3 워크드 예시 (가정, 검증 전)
> 가정: 월 1,000통화 × 평균 5분 = **5,000분**, 양방향 브리지(착신+합성 leg 2배 분).

| 항목 | 단가(가정) | 수량 | 소계(월) |
|------|-----------|------|---------|
| 종단(착신) | $0.005/min | 5,000분 | $25 |
| 발신/합성 leg | $0.007/min | 5,000분 | $35 |
| 번호 월정액 | $1/번호 | 1 | $1 |
| 미디어/통역 컴퓨트 | (RTX 5090 자가호스팅) | — | ≈0 한계비용 |
| **합계(통신비)** | | | **≈$61/월** |

→ **통신 한계비용은 작다**(분당 $0.012 수준 가정 시 5분 통화 $0.06). **주 비용은 컴퓨트·운영·규제 준수**. 자가호스팅 GPU(RTX 5090)가 통역 컴퓨트 한계비용을 크게 낮춘다.

> ⚠️ 모든 단가는 **가정/공개 시작가**다. 실제 견적(특히 한국 번호 종단·번호 월정액·CPS)은 공급사 견적으로 대체할 것.

## 4. 품질·지연 고려 (이미 측정된 자산 활용)

- **지연 예산**: mouth-to-ear P95 < 2s(ITU-T G.114). 서버 브리지는 인앱 대비 **네트워크 1홉 추가**(통신사↔미디어서버) → 지연 가산 측정 필요.
- **동시통역 정책**: wait-k/LA/AlignAtt로 첫 토큰 지연 최소화([`SELF_EVOLVING_ENGINE_DESIGN.md`](SELF_EVOLVING_ENGINE_DESIGN.md) §4.3).
- **코덱/지터/에코**: PSTN/셀룰러 leg는 협대역(G.711/AMR) 가능 → STT 품질 영향. 에코는 서버 AEC 필요(인앱 G10 자산과 별개 경로).
- **평가 연동(T4)**: 전화 채널 통화도 `eval/worldlinco/` 하니스에 **채널 태그**로 편입해 J·R-factor를 동일 척도로 비교(이미 P0/P1/P2 준비됨).

## 5. 규제·개인정보 게이트 (T3 선행 점검)

- **통역/녹음 고지·동의**: 통화 통역은 제3자(엔진) 처리 + 음성/전사 생성. **양 당사자 고지·동의** 흐름 필요. 안내멘트·동의 IVR 검토.
- **개인정보**: 음성·전사본은 민감정보. **저장 최소화·보존기간·삭제권**([`SECURITY_STRIDE_DESIGN.md`](SECURITY_STRIDE_DESIGN.md) 연계). Session Core/Turn 기록은 이미 **오디오 미보관·텍스트 요약만** 원칙.
- **사업자 등록 의무**: 임대 모델이라도 "전기통신역무 제공" 해당 여부 법무 확인(확인 필요).
- **국제 통화**: 관할별 통신감청·동의 법규 상이 → 국가별 검토.

## 6. Go / No-Go 기준 (T1 진입 게이트)

| # | 기준 | Go 조건 |
|---|------|---------|
| G-T0-1 | 번호 모델 확정 | 070(국내)+CPaaS DID(해외) 공급사 1곳 이상 견적 확보 |
| G-T0-2 | 트렁크 임대 가능 | 직접 등록 없이 SIP 트렁크 임대 경로 확인(통신사 또는 CPaaS) |
| G-T0-3 | 비용 수용 | 통화당 통신 한계비용 ≤ 목표(예: $0.10/5분) + 컴퓨트 자가호스팅 확인 |
| G-T0-4 | 규제 클리어 | 고지·동의 흐름 설계 가능 + 등록 의무 법무 1차 의견 확보 |

**4개 모두 Go → T1(미디어 서버 1콜 PoC) 착수.** 하나라도 미충족 → 해당 항목 추가 조사 또는 경로 A(데모 한정)로 한정.

## 8. 권고 (T0 결론)

1. **경로 B로 수렴, 단 임대 모델로 시작** — 직접 기간통신 등록(자본금 3억/30억·인력·보호기구)은 초기 부적합.
2. **번호: 070+CPaaS DID** 조합으로 국내·해외 동시 커버.
3. **비용 병목은 통신비가 아니라 컴퓨트·운영·규제** — 자가호스팅 GPU로 통역 한계비용 최소화.
4. **규제·동의는 T3가 아니라 T0에서 1차 법무 의견을 받아 리스크를 조기 가시화**.
5. PoC는 경로 A(인앱 음향 브리지, 코어 재사용)로 **즉시** 데모 가능 — 사업 결정과 병행.
