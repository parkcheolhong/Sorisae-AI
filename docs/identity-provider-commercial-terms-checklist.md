# 본인확인 상용화 기준 계약·약관 체크리스트

## 목적

- `PASS`, `KMC`, `KCB` 공급사와 실제 상용 운영 계약을 진행할 때 필요한 계약/약관 기준을 한 문서로 고정합니다.
- 관리자 화면에서 바로 열어 운영/법무/정산 체크를 동시에 진행할 수 있는 상용화 기준 문서입니다.

---

## 1. 상용화 필수 문서

| 구분 | 개인사업자 | 법인사업자 | 비고 |
|---|---|---|---|
| 사업자등록증 | 필수 | 필수 | 최신본 제출 |
| 서비스 소개서 | 권장 | 필수 | 본인확인 목적, 흐름, 이용자 대상 포함 |
| 개인정보처리방침 | 필수 | 필수 | 본인확인 데이터 처리 목적 명시 |
| 이용약관 | 권장 | 필수 | 본인확인, 계정 복구, 고지 문구 포함 |
| 정산 계좌/세금계산서 정보 | 필수 | 필수 | 과금/정산 담당자 정보 포함 |
| 담당자 연락처 | 필수 | 필수 | 운영/보안 담당자 분리 권장 |
| 서비스 도메인/URL | 필수 | 필수 | callback 등록 대상과 일치 필요 |

---

## 2. 이용약관 필수 조항

### 사용자 약관에 반드시 들어갈 항목

1. 본인확인 서비스 사용 목적
2. 본인확인 실패/재시도 정책
3. 계정 복구 및 비밀번호 재설정 절차
4. 본인확인 공급사 연동 사실 고지
5. 본인확인 결과에 따른 서비스 제한 가능성
6. 부정 사용 및 타인 명의 사용 금지
7. 운영 중단/점검/장애 시 고지 기준

### 권장 문구 체크

- `회원 본인확인 및 계정 보호를 위해 외부 본인확인 기관(PASS/KMC/KCB 등)을 이용할 수 있습니다.`
- `본인확인 실패 또는 정보 불일치 시 일부 기능 이용이 제한될 수 있습니다.`
- `비밀번호 재설정, 고위험 작업, 민감 정보 변경 시 추가 본인확인을 요구할 수 있습니다.`

---

## 3. 개인정보처리방침 필수 조항

| 항목 | 필수 여부 | 설명 |
|---|---|---|
| 수집 목적 | 필수 | 회원 확인, 계정 보호, 비밀번호 재설정 |
| 수집 항목 | 필수 | 이름, 생년월일, 휴대전화, CI/DI 등 공급사 제공값 |
| 보유 기간 | 필수 | 법령/내부 정책 기준 명시 |
| 제3자 제공 | 필수 | 공급사/위탁 처리 여부 명확화 |
| 파기 절차 | 필수 | 인증 완료 후 저장 정책 또는 즉시 파기 기준 |
| 이용자 권리 | 필수 | 열람, 정정, 삭제, 처리정지 요청 절차 |

---

## 4. 계약서 확인 항목

| 항목 | 확인 내용 |
|---|---|
| 성공 과금 | 성공 건만 과금인지 확인 |
| 시도 과금 | 시도 시 과금되는지 확인 |
| 실패 과금 | 실패/오류 건 과금 여부 확인 |
| 재시도 과금 | 동일 사용자 재시도 비용 확인 |
| 월 기본료 | 고정 비용 존재 여부 |
| 최소 약정 | 최소 사용량/약정 기간 |
| 테스트 계정 | 무료/유료 여부 |
| callback 등록 | 운영 callback URL 고정 등록 가능 여부 |
| SLA | 장애 대응 시간, 운영 지원 시간 |
| 보안 요구 | IP 제한, VPN, 망분리, 키 교체 주기 |

---

## 5. 운영 전 승인 게이트

상용화 전 아래 조건을 모두 만족해야 합니다.

- [ ] 관리자 화면에 공급사 운영값 저장 완료
- [ ] request 매핑 카드 정상
- [ ] complete 매핑 카드 정상
- [ ] callback URL 실서버 등록 완료
- [ ] 개인정보처리방침 공개 URL 준비 완료
- [ ] 이용약관 공개 URL 준비 완료
- [ ] 과금/정산 조건 문서 확인 완료
- [ ] 실패/재시도 과금 기준 상품 원가에 반영 완료
- [ ] 실제 관리자 비밀번호 재설정 흐름 검증 완료
