## 3.3 VoIP 통역 통화 — P1(시그널링)
패키지 `backend/voip/`: `config / models / registry / signaling / router (/ redis_backend)`. `backend/main.py` 라우터 등록.

**REST 계약**
- `POST /api/v1/voip/calls/initiate` (Bearer) → `{call_id, signaling_server(완전 ws URL), turn_servers, participant_role, status, ...}`. 앱 타깃 없으면 PSTN 다이얼러 폴백.
- `GET /api/v1/voip/calls/{call_id}/audit` (Bearer) → 상태/참가자/감사 이벤트.
- `POST /api/v1/voip/calls/{call_id}/end` (Bearer) → 종료 + hangup 통지.

**WebSocket**: `WS /api/v1/voip/ws/{call_id}?token=<JWT>&role=<caller|callee>`. 릴레이 타입 `offer/answer/candidate/chat_message/voice_translation` + `ping`→`pong` + `hangup`. **앱↔앱 자동매칭**(양측 initiate → 같은 call_id, role 구분).

**전제(P1)**: 앱↔앱 · STUN+env TURN · 인메모리 단일 워커. 미디어는 P2P+TURN(서버 미중계).

## 3.4 VoIP — P2(Redis 스토어 + pub/sub 릴레이)
- `backend/voip/redis_backend.py`: `RedisCallStore`(룸/감사 Redis, 매칭 인덱스 `voip:incoming:{uid}`) + `RedisRelay`(채널 `voip:relay:{call_id}` pub/sub) → **멀티 워커 지원**.
- 팩토리 `get_store()/get_relay()` — `VOIP_REDIS_URL` 유무로 Redis/인메모리 선택. 단일/멀티워커 **동일 코드경로**.
- 서버측 chat 번역: 모바일이 클라이언트에서 수행하므로 P2 제외(문서화).

## 6.2 P3-A 상세 (구현 완료)
- **presence.py**: 사용자별 FCM 디바이스 토큰 집합 + presence(온라인) TTL 키. `VOIP_REDIS_URL` 시 Redis(멀티워커), 아니면 인메모리. `VOIP_PRESENCE_TTL_SEC`(기본 120), `VOIP_DEVICE_TTL_SEC`(기본 30일).
- **push.py**: FCM 어댑터. `FCM_ENABLED=true` + (`GOOGLE_APPLICATION_CREDENTIALS` 또는 `FCM_CREDENTIALS_JSON`) 구성 시 `firebase-admin`으로 멀티캐스트 data 푸시(android high priority), 미설치/미구성 시 `skipped` no-op. `firebase-admin`은 선택 의존성(요건 미포함).
- **API**: `POST /api/v1/voip/devices/register {fcm_token, platform?}`(Bearer) → 토큰 저장 + presence 온라인. `initiate`(caller 턴)에서 콜리 presence로 `callee_app_online` 산정 + 콜리 토큰에 착신 푸시 + 감사(`push_sent`/`push_skipped`). WS 접속 시 토큰 uid로 presence 갱신.
- **검증**: `test_voip_presence_push.py`(등록/푸시 호출/오프라인 skip/미구성 skip) 4 passed. 라이브: register `ok`, initiate `callee_app_online=True`(등록 직후), 푸시는 FCM 미구성으로 skip.
- **남은 연동**: 모바일 측 FCM 토큰 등록 호출 + 착신 data 푸시 수신→`initiate`(자동매칭) 또는 accept 합류 + Firebase 초기화(이전 `No Firebase App` 오류 해소).

## 6.1 P3-C / voice-translate 상세 (구현 완료)
- **TURN 토큰**: `VOIP_TURN_STATIC_AUTH_SECRET` 설정 시 `username="<expiry>:<user_key>"`, `credential=base64(HMAC-SHA1(secret,username))`, TTL `VOIP_TURN_TOKEN_TTL_SEC`(기본 86400). `get_ice_servers(user_key)`가 통화/사용자별 동적 자격 발급, 미설정 시 정적 `VOIP_TURN_USERNAME/CREDENTIAL` 폴백. initiate가 `user_key=caller_user_id`로 호출.
- **voice-translate**: `POST /api/llm/voice-translate {audio_base64|transcript, from_lang, to_lang, region_hint?, tts?}` → STT(whisper.cpp→faster-whisper 폴백, from_lang 힌트) → `NadoTranslator.translate` → `{original_text, translated, engine:"nado", detected_language, audio_base64?, audio_format?}`(모바일 `VoiceTranslateResult`와 일치). 모바일 `voiceTranslate`/VoIP 음성 릴레이가 이제 실동작.
- **동시성 수정(멀티워커 정합성)**: RedisCallStore를 read-modify-write(룸 JSON 통째 저장)에서 **HASH(상태/참가자)+LIST(이벤트 RPUSH)** 로 전환, 매칭은 **SPOP**으로 원자화, 상태는 **Lua 단조 전이(ringing<connecting<connected<ended, ended 종단)**. WS는 `register`(구독) 후 `accept`, hangup은 `end()` 후 상대 통지 → 이벤트/상태 유실 제거.
