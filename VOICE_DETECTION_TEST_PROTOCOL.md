# 음성 인식 언어 자동 인식 테스트 프로토콜

## 목표
음성 입력 시 **한글 국가명/언어명**이 올바르게 인식되어 **언어 패널이 자동으로 전환**되는지 확인.

---

## 테스트 환경 설정

### 사전 준비
1. **기기**: Samsung Galaxy (R3CT209943N 또는 동등)
2. **앱**: 나도통역사 (최신 APK 설치)
3. **백엔드**: <http://127.0.0.1:8000> 실행 상태
4. **네트워크**: 기기가 PC와 같은 네트워크 또는 USB 디버깅 활성화
5. **마이크**: 정상 작동 확인

### 앱 준비

```
1. 나도통역사 앱 실행
2. 하단 "음성 번역" 탭 선택
3. "마이크" 또는 "🎤" 버튼 확인 (녹음 모드)
```

## 테스트 케이스

### **테스트 1: "미국" (영어)**

| 항목 | 값 |
|------|-----|
| **음성 입력** | "미국" (또는 "미국으로 번역해줘") |
| **예상 동작** | 1. 음성 인식 완료 후 언어 패널이 **English → Korean**으로 자동 전환 |
| | 2. 소스 언어(Source): **EN** (English) |
| | 3. 타겟 언어(Target): **KO** (Korean) |
| **실패 기준** | • 패널이 Korean → English로 유지됨 |
| | • 언어가 자동 변경되지 않음 |
| | • 소스 언어가 Korean으로 표시됨 |

**로그 확인** (PC 터미널):

```bash
# 백엔드 컨테이너 로그
docker logs -f devanalysis114-backend | grep -i "미국\|english\|source.*en"
```

**검증 체크리스트**:
- [ ] 음성 입력 후 텍스트가 "미국"으로 표시됨
- [ ] 언어 패널 자동 변경 확인 (0.5초 이내 반응)
- [ ] 소스 언어가 EN으로 표시됨
- [ ] 타겟 언어가 KO로 표시됨

## 로그 저장 및 분석

### 백엔드 타이밍 로그

```bash
# transcribing vs translating 비교
docker logs devanalysis114-backend 2>&1 | grep "TRANSCRIBING\|TRANSLATING\|TOTAL="
```

**출력 예시**:

```
[JOB abc123] TRANSCRIBING=2.34s (segments=5)
[JOB abc123] TRANSLATING=3.56s (segments=5)
[JOB abc123] TOTAL=5.90s | TRANSCRIBE=2.34s | TRANSLATE=3.56s | SEGMENTS=5
```

### 결과 분석

```
TRANSLATING > TRANSCRIBING인 경우:
→ 세그먼트 캐싱이 효과적 (반복 가사 있음)
→ 추가 최적화: googletrans 배치 요청 또는 병렬 처리

TRANSCRIBING > TRANSLATING인 경우:
→ 음성 인식이 병목 (faster-whisper 자체 성능)
→ 추가 최적화: GPU 가속 또는 모델 경량화
```
